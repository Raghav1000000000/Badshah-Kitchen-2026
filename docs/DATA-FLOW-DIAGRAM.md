## Database Query Optimization

### Before Indexes
```sql
-- Kitchen Query (100-200ms)
SELECT * FROM orders WHERE status != 'COMPLETED'
ORDER BY created_at DESC;
-- Full table scan on both conditions

-- Orders Query (50-100ms)
SELECT * FROM orders WHERE session_id = 'abc-123'
ORDER BY created_at DESC;
-- Full table scan on session_id
```

### After Indexes
```sql
-- Kitchen Query (20-30ms)
SELECT * FROM orders WHERE status != 'COMPLETED'
ORDER BY created_at DESC;
-- Uses: idx_orders_status_created (composite index)

-- Orders Query (10-20ms)
SELECT * FROM orders WHERE session_id = 'abc-123'
ORDER BY created_at DESC;
-- Uses: idx_orders_session_id + idx_orders_created_at

Performance Improvement: ~5x faster! ğŸš€
```
